Application Architecture

Project Structure

lib/

├── config/         # App configuration and constants

├── data/           # Data layer

│   ├── models/     # Data models

│   ├── repositories/ # Repository implementations

│   └── services/   # External services

├── presentation/   # UI layer

│   ├── screens/    # App screens

│   ├── widgets/    # Reusable UI components

│   └── state/      # State management

└── utils/          # Utility functions

Architectural Pattern

The app follows Clean Architecture principles with:

  1. Data Layer - External data sources and repositories
  2. Domain Layer - Business logic and use cases
  3. Presentation Layer - UI components and state management

Authentication and Security

  • Authentication: Managed through Supabase Auth with email/password, Google OAuth, and Apple Sign-In
  • Row Level Security (RLS): Database security implemented at the Supabase level
  • Secure Storage: Sensitive information stored using flutter_secure_storage
  • Environment Variables: API keys and sensitive configuration managed through .env files (not committed to version control)

Additional Development Notes

  1. Run flutter pub run build_runner build --delete-conflicting-outputs after making changes to model files to regenerate freezed and JSON serialization code.
  2. Use .env file for environment variables and ensure it's added to .gitignore.
  3. Follow the repository pattern for data access to maintain a clean separation of concerns.
  4. Implement comprehensive error handling, especially for network requests and audio playback.
  5. Ensure background audio continues to work when the app is minimized or the screen is locked.
