Font Configuration

Base Font (Inter)

  • Inter Light (300) — Primary body text
  • Inter Regular (400) — UI elements, buttons
  • Inter Medium (500) — Emphasis, interactive elements

Text Sizes

  • Body text: 16px / fontSize: 16.0
  • Secondary text / labels: 14px / fontSize: 14.0
  • Small text / captions: 12px / fontSize: 12.0
  • Line height: 1.5–1.65 (use height: 1.5 in Flutter)

Headers (Cormorant Garamond)

  • H1 (App title / major section): 28–32px / fontSize: 28.0
  • H2 (Section titles): 20–24px / fontSize: 22.0
  • H3 (Subsection headers): 18px / fontSize: 18.0

Use slightly increased letter spacing (0.5-1.0) for headers to enhance elegance.

Logo Typography

  • Use the Mourier font specifically for the letter "O" in the app logo
  • Keep as the only brand mark—minimal and memorable
  • Can be animated gently on splash screen or as a breathing loop

📐 Spacing & Layout Rules

Grid System

Implement an 8-point base grid throughout the application:

  • Base unit: 8dp
  • Common spacings: 8dp, 16dp, 24dp, 32dp, 40dp, 64dp
  • Button/interactive element height: 48dp minimum (for comfortable tapping)
  • Input field height: 48-56dp

Screen Structure

  • Screen padding: 16dp left/right
  • Section padding: 24dp between major content sections
  • Bottom navigation height: 56dp
  • Safe area: Always respect device safe areas

Component Spacing

  • Card padding: 16-24dp inside cards, modals, etc.
  • List item padding: 16dp vertical, 16dp horizontal
  • Form field spacing: 24dp between fields
  • Icon + text spacing: 8dp between icon and associated text

Margins

  • Above titles/headers: 24-32dp
  • Between grouped elements: 8-16dp
  • Before/after dividers: 16dp

🌟 Modern Design Elements

Glassmorphism Components

Glassmorphism creates a frosted glass effect with transparency, blur, and subtle borders, providing depth and layering while maintaining the minimal aesthetic.

1. GlassContainer

  • Apply to overlays, modals, and informational cards
  • Creates depth without visual heaviness
  • Preserves context by showing blurred content beneath

// Glass effect container

class GlassContainer extends StatelessWidget {

  final Widget child;

  final double blur;

  final double opacity;

 

  const GlassContainer({

    Key? key,

    required this.child,

    this.blur = 10.0,

    this.opacity = 0.2,

  }) : super(key: key);

 

  @override

  Widget build(BuildContext context) {

    return ClipRRect(

      borderRadius: BorderRadius.circular(16.0),

      child: BackdropFilter(

        filter: ImageFilter.blur(sigmaX: blur, sigmaY: blur),

        child: Container(

          decoration: BoxDecoration(

            color: Colors.white.withOpacity(opacity),

            borderRadius: BorderRadius.circular(16.0),

            border: Border.all(

              color: Colors.white.withOpacity(0.2),

              width: 1.5,

            ),

          ),

          child: child,

        ),

      ),

    );

  }

}

Neumorphism Components

Neumorphism creates a soft, extruded plastic look that adds tactile quality to interactive elements while maintaining the clean aesthetic.

1. NeumorphicButton

  • Apply to primary action buttons
  • Creates a sense of physical interaction
  • Provides subtle feedback through shadow changes

// Neumorphic button

class NeumorphicButton extends StatelessWidget {

  final Widget child;

  final Function() onPressed;

  final bool isPressed;

 

  const NeumorphicButton({

    Key? key,

    required this.child,

    required this.onPressed,

    this.isPressed = false,

  }) : super(key: key);

 

  @override

  Widget build(BuildContext context) {

    // Use colors from your palette

    final baseColor = Color(0xFFF7F7F7); // Your Light Gray

   

    return GestureDetector(

      onTap: onPressed,

      child: AnimatedContainer(

        duration: AnimDuration.medium,

        curve: AnimCurve.gentle,

        padding: EdgeInsets.symmetric(

          vertical: Spacing.m,

          horizontal: Spacing.l,

        ),

        decoration: BoxDecoration(

          color: baseColor,

          borderRadius: BorderRadius.circular(12),

          boxShadow: isPressed

              ? [

                  // Inner shadow when pressed

                  BoxShadow(

                    color: Color(0xFF2F4858).withOpacity(0.1),

                    offset: Offset(2, 2),

                    blurRadius: 2,

                    spreadRadius: 1,

                  ),

                  BoxShadow(

                    color: Colors.white.withOpacity(0.7),

                    offset: Offset(-2, -2),

                    blurRadius: 2,

                    spreadRadius: 1,

                  ),

                ]

              : [

                  // Outer shadow when not pressed

                  BoxShadow(

                    color: Color(0xFF2F4858).withOpacity(0.1),

                    offset: Offset(4, 4),

                    blurRadius: 6,

                    spreadRadius: 1,

                  ),

                  BoxShadow(

                    color: Colors.white,

                    offset: Offset(-4, -4),

                    blurRadius: 6,

                    spreadRadius: 1,

                  ),

                ],

        ),

        child: child,

      ),

    );

  }

}

Where to Apply Neumorphism:

  • Primary Action Buttons: "Begin Meditation," "Start Session"
  • Media Controls: Play/pause, volume, and timer controls
  • Bottom Navigation Items: For subtle tactile feedback
  • Interactive Cards: Artist selection tiles or track selection

Design Element Integration Guidelines

  1. Selective Application
    • Apply these effects to only 2-3 component types in the entire app
    • Keep the majority of UI using established flat design
    • Reserve these effects for focal points and interactive elements
  2. Consistency Rules
    • Maintain consistent light source direction (top-left) for all neumorphic elements
    • Use consistent blur values for all glassmorphism instances
    • Don't mix these effects on the same component
  3. Subtlety is Key
    • Use low opacity (0.1-0.2) for glass effects
    • Keep shadows light for neumorphic elements (avoid heavy 3D effects)
    • Maintain soft transitions between states
  4. Context Appropriate
    • Use glassmorphism for informational/passive elements
    • Use neumorphism for interactive elements
    • Consider which effect best supports each component's function

🌬️ Motion & Transitions

Animation Principles

  • Gentle: All animations should feel calm and intentional
  • Slow: Use longer durations (300-800ms) for most transitions
  • Natural: Employ natural easing curves (ease-out, ease-in-out)

Standard Animations

  1. Screen Transitions
    • Fade transitions between screens (duration: 300-500ms)
    • Subtle slide-up for modal screens (with ease-out curve)
  2. Micro-interactions
    • Button press: Subtle scale down (0.98) with quick spring back
    • Success indicators: Gentle fade in with slight scale up
  3. Meditation Elements
    • Breathing visualization: Slow, rhythmic scaling (4-6 seconds per cycle)
    • Progress indicators: Smooth, continuous motion
    • Timer countdown: Subtle pulse on minute changes

Implementation Guidelines

📱 Responsive Design

The meditation app primarily targets mobile devices but should gracefully adapt to different screen sizes:

Target Breakpoints

  • Small phones: 320dp width minimum
  • Standard phones: 360-400dp
  • Large phones: 400-480dp
  • Tablets: 600dp+

Layout Adaptation

  • Use MediaQuery to determine screen size and adjust layout accordingly
  • On tablets, consider using a multi-column layout where appropriate
  • Maintain minimum touch target size of 48dp across all devices
  • Use SafeArea to respect device notches and system UI

Implementation Tips

// Helper for responsive design

class Responsive {

  static bool isTablet(BuildContext context) {

    return MediaQuery.of(context).size.shortestSide >= 600;

  }

 

  static double getCardWidth(BuildContext context) {

    final width = MediaQuery.of(context).size.width;

    if (width > 600) {

      return width * 0.4; // 40% of screen on tablets

    } else {

      return width - 32; // Full width minus padding on phones

    }

  }

}

🔍 Accessibility Guidelines

Ensure the meditation app is accessible to all users:

  1. Text Contrast: Maintain WCAG AA standard (4.5:1) for all text
  2. Font Sizing: Support dynamic text sizing for users with vision needs
  3. Tap Targets: Keep all interactive elements at least 48dp tall/wide
  4. Screen Readers: Implement proper semantics and content descriptions
  5. Alternative Inputs: Support keyboard navigation where applicable

⚠️ Additional Principles

  • Avoid high-contrast or saturated colors
    • No neon, bright reds, or jarring tones
    • Keep UI elements soft with muted accents and neutral space
  • Minimize visual noise
    • Each screen should focus on one primary action or state
    • Use progressive disclosure for complex features
    • Avoid excessive decoration or ornamental elements
  • Create breathing room
    • Don't fill every available space with content
    • Use whitespace intentionally to create a sense of calm
    • Allow UI elements room to "breathe"
